# PC Wrapped — Design Spec

**Дата:** 2026-06-21
**Статус:** Утверждён (готов к написанию плана реализации)
**Рабочее название:** PC Wrapped

## 1. Обзор

Фоновое десктоп-приложение под Windows на Avalonia C#. Тихо живёт в системном
трее, локально собирает статистику использования ПК и в любой момент (плюс
авто-итоги недели/года) генерирует красивую карточку-отчёт, которую одной кнопкой
экспортируешь в PNG (квадрат 1:1 и вертикаль 9:16) и постишь в соцсети.

### Цель продукта
Бесплатный «wow»-инструмент с встроенной механикой распространения: каждая
расшаренная карточка — это реклама приложения. Ниша красивого локального
«Wrapped для ПК» под Windows практически пуста (RescueTime — платная веб-подписка),
что даёт шанс занять её.

### Ключевой принцип — приватность
Все данные хранятся **только локально**, ничего не уходит в сеть. Это этично,
снимает страх «следилки» и является сильным аргументом в посте.

### Целевая аудитория
Массовая, через хайп в соцсетях (Reddit, X, Telegram). Формат «Spotify Wrapped»
люди любят и постят сами.

## 2. Метрики (что показываем в карточке)

- **⏱️ Время по приложениям/играм** — топ по времени. Через опрос активного окна
  (безопасно, без хуков).
- **🎮 Категории** — работа / игры / соцсети / браузер. Авто-классификация по
  встроенному словарю + ручные правки. Круговая диаграмма «на что ушло время».
- **📈 Ритм/продуктивность** — самый продуктивный час, тепловая карта активности
  по часам/дням, streak-серии.
- **🖱️⌨️ «Vanity»-цифры** — километры пробега мыши, число кликов и нажатий клавиш.
  **Только счётчики, никогда не содержимое набранного.**

### Замечание по vanity-метрикам и доверию
Подсчёт нажатий/мыши требует низкоуровневых системных хуков ввода (тех же, что у
кейлоггеров) → антивирусы могут реагировать. Решение:
- Функция включена, но с честным объяснением в UI (что именно считается и что
  содержимое не сохраняется).
- **Открытый исходный код** проекта — любой может убедиться, что текст не пишется.
  Открытость сама по себе вирусный аргумент доверия и дополнительный канал
  распространения.
- Мягкая деградация: если хук недоступен/заблокирован — приложение продолжает
  считать время приложений, vanity-метрики показывают «недоступно», без краша.

## 3. Темы карточки

Три выбираемые в приложении темы (система тем data-driven, легко добавлять новые):

- **A — Яркий градиент** (Spotify-Wrapped вайб; массовая аудитория)
- **B — Dev / неон-терминал** (тёмная техно-эстетика; айтишники, геймеры)
- **C — Минимализм** (чистый светлый дата-виз; продуктивити-аудитория)

Каждая тема рендерится в двух форматах экспорта: **1:1** (лента) и **9:16** (сторис)
из одних и тех же данных.

## 4. Архитектура и компоненты

Изолированные части с чёткими границами, каждая тестируется отдельно.

- **Tracker (сборщик)** — фоновый сервис. Раз в ~1–2 сек опрашивает активное окно
  (`GetForegroundWindow` → процесс/заголовок). Отдельно слушает низкоуровневые хуки
  ввода (`WH_KEYBOARD_LL` / `WH_MOUSE_LL`) и копит **только счётчики** (нажатия,
  клики, пиксели мыши). Учитывает простой (idle) — не считает время отсутствия.
- **Storage** — локальная SQLite (`Microsoft.Data.Sqlite`). Сырые сэмплы активности
  + агрегаты. Интерфейс `IStatsRepository`.
- **Categorizer** — сопоставляет имя процесса → категория. Встроенный словарь +
  пользовательские переопределения.
- **Aggregator** — превращает сырьё в метрики периода (топ-приложений, пик-час,
  тепловая карта, км мыши, streak). Чистые функции → легко тестировать.
- **Card Renderer** — берёт метрики + выбранную тему и рисует карточку средствами
  Avalonia, экспорт в PNG (1:1 и 9:16). Темы подключаемые (data-driven).
- **UI Shell** — окно приложения: дашборд со статистикой, превью карточки, выбор
  темы/периода, кнопка «Поделиться», настройки (что трекать, автозапуск, исключения).
- **Tray + Autostart** — иконка в трее, тихий автозапуск с Windows.

### Поток данных

```
Tracker → Storage (сырьё) → Aggregator (метрики периода) → Card Renderer (PNG) → шеринг
```

Дашборд и карточка читают из Aggregator, не из сырья напрямую.

## 5. Стек

- .NET 8 + Avalonia, C#
- SQLite (`Microsoft.Data.Sqlite`)
- Низкоуровневые хуки — P/Invoke (`user32.dll`), без тяжёлых зависимостей
- Трей-иконка — Avalonia `TrayIcon`

## 6. Надёжность / краевые случаи

- Хук ввода падает/блокируется антивирусом → время приложений продолжает считаться,
  vanity-метрики мягко деградируют («недоступно»), без краша.
- Многомониторность и DPI → расстояние мыши в пикселях с учётом масштаба.
- Простой / сон / блокировка экрана → время не накручивается.
- Рост БД → сырьё старше N дней сворачивается в часовые агрегаты.
- Первый запуск без данных → дружелюбный пустой стейт.

## 7. Дистрибуция и онбординг

- Один установщик / portable .exe, без аккаунтов и регистрации.
- Первый экран — честный экран о приватности + тумблеры «что трекать».
- Открытый исходник на GitHub (доверие + канал распространения).

## 8. Тестирование

- Юнит-тесты на чистую логику: Aggregator (метрики, streak, пик-час),
  Categorizer, конвертация пикселей→км.
- Card Renderer: генерация PNG без падений для всех тем + проверка размеров
  (1:1, 9:16).
- Storage: интеграционные тесты на in-memory SQLite.
- Tracker/хуки — за интерфейсом, мокаются; ручная проверка живьём.
